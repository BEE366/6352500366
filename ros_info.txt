หน้าที่ของ ROS master เป็น Server ให้ node หลายตัวเชื่อมต่อกัน สามารถส่งข้อความหากันได้

การทำงานของ ROS Topics, ROS Service
    ROS Topics คือ ข้อความจะถูกส่งผ่านระบบการขนส่งที่มีการ publish/subscribe
         โหนดส่งข้อคือ publishing ชื่อที่ขึ้น Topic คือเป็นหัวข้อของเนื้อหา
     ROS Service คือการกำหนดการส่งข้อความ ซึ่งจะมีข้อความร้องขอและการตอบกลับ

ความแตกต่างของ Topic กับ message
   message คือ node จะคุยกันผ่าน Message
   Topic คือ จะส่ง message ขึ้นไปที่ Topic จะมี subscriber node มารอรับ message ผ่านทาง Topic ที่มีชื่อเดียวกัน

ความแตกต่างในเชิงการใช้งาน ROS Topic และ Ros service
Topic ใน ROS เป็นการเผยแพร่การสื่อสาร สร้างความสะดวก ข้อความที่เผยแพร่ในหัวข้อสมาชิกทั้งหมดจะเห็น
Service ใน ROS จะเป็นการมีการส่งการคำตอบสำหรับการร้องขอ ซึ่งจะเป็นการสังสารแบบซึงใครนิส
